eval的使用

eval是个函数，可以将字符串转换成代码并立即执行！

使用eval处理JSON格式数据的时候的问题

当使用eval处理JSON格式数据的时候，eval会将JSON格式字符串中的大括号{}，当做代码段来解析，所以会报错！
解决方案：

  1. 将变量声明也同时放在eval中

  2. 在json格式的字符串前后拼接小括号

Function和eval的异同

  1. 都能将字符串转换成代码
  2. Funciton创建出来的是函数，需要手动去调用，代码才会执行，而eval会直接将字符串转成代码并执行

问题：

  1. 执行效率的问题
  2. 安全性问题（XSS）

函数的原型链

如果将函数当做是一个对象的话，那么他也是由构造函数创建出来的，所以这个函数对象，也有自己的原型链
js中所有函数的构造函数都是Function ,包括Function 本身。

Object 和 Function 的关系

可以通过 instanceof 关键字来确定两者之间的关系，Object 和 Function 互为实例。

arguments 对象。

arguments 是函数内部的一个对象，是一个伪数组。
在函数调用的时候，会将函数的实参，依次的存入这个伪数组中。

arguments.length 属性，表示函数调用的时候，传入的实参的个数。
arguments.callee 属性，表示arguments对象所在的函数，一般用来在匿名函数中做递归。

函数对象的成员

  • arguments
  • caller 函数的调用环境，如果在全局，为null ,如果是在其他函数中调用， 那么caller为所在的那个函数
  • length 函数形参的个数
  • name 函数的名称 函数声明和函数表达式的函数名为正常的函数名， new Function() 创建出来的函数名为anonymous

静态成员以及实例成员

  • 静态成员： 通过构造函数访问的成员就是静态成员
  • 实例成员： 通过实例访问的成员就是实例成员。
